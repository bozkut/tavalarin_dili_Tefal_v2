TASK 1: IMPROVE generate_letter_scratches.py (Synthetic Generation)
STATUS: DONE
================================================================================

OBJECTIVE:
  Rewrite letter rendering from font text (which Claude doesn't recognize)
  to hand-drawn stroke simulation, plus improve background scratch patterns
  to avoid false letter detection.

ROOT CAUSE OF FAILURE:
  Previous v1.0 approach used ImageFont.text() to render white letters directly
  on pan background. This produced printed text appearance that Claude's vision
  model (trained on organic patterns) couldn't recognize. Only 1/78 detections
  were successful.

================================================================================
IMPLEMENTATION COMPLETE: 4 MAIN CHANGES
================================================================================

1. NEW FUNCTION: letter_contours_to_strokes()
   Location: Lines 98-174
   Purpose:  Extract letter contours from rendered glyph using OpenCV
   Details:
   - Renders letter on temporary 400x400 image
   - Uses cv2.findContours() to extract all contours
   - Simplifies with Douglas-Peucker algorithm (cv2.approxPolyDP)
   - Normalizes points to [0,1] range for later scaling
   - Returns list of strokes (each stroke is list of [x,y] tuples)
   - Gracefully handles OpenCV unavailability

2. NEW FUNCTION: draw_strokes_with_imperfections()
   Location: Lines 177-233
   Purpose:  Draw strokes with hand-drawn effects to mimic real scratches
   Details:
   - Multiple passes (2x) with varying thickness and jitter
   - Thickness: 2px to 1px for tapering effect
   - Random jitter: -0.5 to +0.5 pixels per pass
   - Color: light gray (200,200,200) instead of bright white
   - Simulates organic feel of real pan surface scratches

3. REWRITTEN: add_background_scratches()
   Location: Lines 236-293
   Purpose:  Create clustered scratch patterns avoiding letter-like formations
   Changes from v1.0:
   OLD: Random scattered lines (1-8 per image) formed V/N/X patterns
   NEW: Clustered patterns (3-6 clusters, 2-4 scratches per cluster)

   New algorithm:
   - Creates 3-6 random clusters across pan surface
   - Each cluster centers at random location (50-380px from center)
   - 2-4 scratches branch radially from cluster center
   - Angles vary 0-360 degrees to avoid letter formation
   - Lengths vary (30-150px) for natural appearance
   - Gray levels: subtle (50-100) to not interfere
   - Thin lines: 1-2 pixels for natural look
   - Applied Gaussian blur (radius=0.3) for soft edges

4. UPDATED: draw_letter_scratches()
   Location: Lines 296-374
   Purpose:  Main letter rendering with dual strategy
   Changes:
   - Strategy 1 (preferred): Uses OpenCV stroke rendering if available
     * Calls letter_contours_to_strokes()
     * Calls draw_strokes_with_imperfections()
     * Hand-drawn appearance with jitter
   - Strategy 2 (fallback): PIL font rendering
     * Changed color from bright white (245,245,245) to light gray (200,200,200)
     * Single pass instead of multiple overlays
     * Applied Gaussian blur (radius=0.5) to soften edges
   - Graceful fallback chain for font loading (Arial, DejaVuSans, Liberation)

5. UPDATED: generate_letter_image()
   Location: Line 304
   Change: Background scratch count
   OLD: random.randint(1, 8)
   NEW: random.randint(10, 20)
   Rationale: More scratches create busier, more realistic background
